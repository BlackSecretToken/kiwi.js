Kiwi.Files.File public

Defined in src/file/Loader.ts:963

Create a new File to store an audio piece.

First check to see if the AUDIO file being loaded is supported by the browser/device before adding it to the loading queue. You can override this behaviour and tell the audio data to load even if not supported by setting the onlyIfSupported parameter to false.

If you pass an array of filepaths as url, the first audio filetype that is supported will be loaded.

Parameters:

  • key String

    Key for the audio file

  • url String

    URL of the audio to load. You can pass an array of URLs, in which case the first supported audio filetype in the array will be loaded.

  • [storeAsGlobal=true] Boolean optional

    Whether the file should be stored globally

  • [onlyIfSupported=true] Boolean optional

    Whether the audio file should only be loaded if Kiwi detects that the audio file is supported

start

(
  • [calculateBytes]
)
public

Defined in src/file/Loader.ts:647

Available since 1.2.0

Start loading all the files which are in the file queue.

To accurately use the bytesLoaded or bytesTotal properties, you will need to set the calculateBytes boolean to true. This may increase load times, as each file in the queue will first make XHR HEAD requests for information.

When calculateBytes is true, percentLoaded equals bytesLoaded / bytesTotal. Otherwise it equals filesLoaded / numberOfFilesToLoad.

Parameters:

  • [calculateBytes] Boolean optional

    Set calculateBytes property on loader

onQueueProgress

Kiwi.Signal public

Defined in src/file/Loader.ts:134

Available since 1.2.0

Dispatches callbacks each time a file in loadingList has been loaded. Callbacks dispatched are passed the following arguments in order.

  1. percent - Percentage of files loaded. A number from 0 - 100
  2. bytesLoaded - Number of bytes loaded
  3. file - Latest file that was loaded. First call will be null.

Make sure to remove callbacks (or use the addOnce method), otherwise will fire again when other sections use the loader.
